What is AYMME?

AYMME stands for "Are You Mocking ME?" and this tool makes mocking data easier when using running your Web Application. Most Javascript Framework already covering the instant feedback when code changes, but when you need to test multiple scenarios using different data, error responses or error messages this can really annoying. AYMME is made in order to manage this for you by providing the following functionalities.

  • Mocking API (backend) responses
  • Mocking different status codes (200, 500, 404 etc.)
  • JSON editor to provide the responses
  • Set specific project configuration to group endpoints
  • Share AYMME projects within your team

Depending on the state of your project, it might not be needed to use these functionalities. But you can always use this tool in a demo to quickly switch to different set of data.

Mocking Back-End Services

Any component or service that requires data, can be mocked locally using the mock providers of generated data modules. These are convenient if you need static data initially. However, if you want to try other scenarios like when it returns an error from the server or use other data. It cannot be easily modified.

AYMME allows you to change the response of each endpoint easily and simulate the live environment by providing the data through an actual HTTP request. Each request that your application is making, can be registered in AYMME by redirecting the requests to the AYMME Back-End Service. From there, AYMME will be registered and ready for mocking the response. AYMME is also able to manage your mock data, by uploading multiple examples, to switch between different mock data on the fly.

How to Configure


          

            The components or services in your project should be making HTTP requests to AYMME now. To verify this,
            select your project in the AYMME web application page. This will list all the requests made by your project
            application on the left side. When your application is performing new requests, simply hit the refresh
            button to fetch the latest endpoints. Now you can start mocking each requests by clicking on the endpoint
            and configure the response on the right bar.
          


          
Managing the Endpoints

          

            Notice that you will have all requests with a collection name called "default". To group your endpoints,
            drag the endpoints to any collection. If the endpoint doens't appear in the list try using the refresh in
            order to fetch new data.
          


          
Managing the Mock Data

          

            When configuring the response data, it might be handy to switch between multiple response. You can have
            multiple responses and switch between them by selecting the active status code.
